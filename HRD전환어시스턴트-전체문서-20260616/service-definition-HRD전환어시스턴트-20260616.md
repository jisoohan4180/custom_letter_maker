# HRD 전환 어시스턴트 서비스 정의서

> 작성일: 2026-06-16
> 버전: v1.0

## 0. Executive Summary (사내 보고용 1페이저)

**한 줄 정의**: HRD 미등록 합격자의 이탈 사유를 AI로 분석해, 문자 발송 담당자에게 과정별 개인화 독려 문자 멘트(HRD 등록 링크 포함)가 담긴 엑셀 파일을 매일 오전 제공하는 내부 운영 웹 도구

**핵심 문제**: 합격 후 HRD 미등록자에게 일괄 리마인드를 보내 전환율이 낮음

**우리 해결책**: 신청자·인터뷰 데이터 기반 이탈 사유 예측 + 개인화 멘트 자동 생성

**차별점**: 기존 일괄 문자 대비 지원자별 고민 요소를 반영한 맞춤 아웃콜 스크립트 제공

**3개월 MVP 범위**: CSV 업로드 → AI 분석 → 엑셀 다운로드 + 과정·멘트 관리 화면

**기대 효과**: 아웃콜 매니저의 사전 준비 시간 단축 + HRD 등록 전환율 개선

**필요 자원**: 개발자 1명 / 3~4주 / Claude API 비용 월 200원 미만(추정)

## 1. 한 문장 서비스 정의

합격 안내 후 2일차 HRD 미등록자의 이탈 사유를 Claude AI로 분석하고, 과정별 맞춤 독려 문자 멘트(HRD 등록 링크 포함)가 담긴 엑셀 파일을 매일 오전 운영 담당자에게 제공하는 내부 웹 도구

---

## 2. 타깃 사용자

### 2.1 1순위 — 운영 담당자 (직접 사용자)
- **누구**: 매일 오전 아웃콜 대상자 파일을 준비하는 운영팀 담당자 1명
- **상황**: 노션에서 미등록 합격자를 확인하고 매니저에게 전달할 파일을 수작업으로 만들고 있음
- **현재 행동**: 노션 데이터를 보고 직접 멘트를 작성하거나 일괄 안내 문구를 복사해서 전달

### 2.2 2순위 — 문자 발송 담당자 (결과물 수령자)
- **누구**: 파일을 받아 지원자에게 개인화 독려 문자를 발송하는 담당자
- **상황**: 지원자별 배경 정보 없이 동일한 안내 문자를 일괄 발송 중
- **현재 행동**: 파일 수령 후 일괄 문자 발송, 개인화 없이 진행

---

## 3. 현재 문제와 기존 대안

### 3.1 문제 정의
합격 안내 후 2일 이내 HRD(내일배움카드로 직업훈련을 신청하는 정부 시스템) 미등록자에게
일괄 리마인드 방식으로 연락하고 있어 전환율에 한계가 있음.
지원자마다 이탈 사유(시간 제약, 과정 확신 부족, 직무 이해 부족 등)가 다름에도
동일한 안내가 나가면서 의사결정 지연 및 이탈로 이어짐.

### 3.2 기존 대안

| 대안 | 한계 |
|---|---|
| 일괄 리마인드 문자/전화 | 개인 고민 요소 미반영, 전환율 낮음 |
| 매니저 수작업 개인화 | 시간이 너무 오래 걸려 일괄 멘트로 대체됨 |
| 노션 AI 활용 | 결과가 노션 안에 머물러 엑셀 전달 워크플로우와 충돌 |

---

## 4. 핵심 가치 제안

**Before**: 미등록자 전원에게 동일한 리마인드 → 개인 고민 미반영 → 이탈

**After**: 지원자별 이탈 사유 예측 → 맞춤 아웃콜 멘트 → 의사결정 지원 → 전환율 개선

핵심 변화: "일괄 연락"에서 "개인 맞춤 의사결정 지원"으로

## 5. 대표 사용 시나리오

### 시나리오 1: 매일 오전 파일 준비
- **상황**: 오전 9시, 운영 담당자가 전날 합격 안내된 미등록자 목록을 노션에서 확인
- **행동**: 노션에서 CSV 내보내기 → 웹 도구에 업로드 → 분석 실행 → 엑셀 다운로드
- **결과**: 기수·이름·번호·이탈예측사유·독려 문자 멘트가 담긴 엑셀을 10분 안에 문자 발송 담당자에게 전달 → 담당자가 지원자에게 개인화 문자(HRD 등록 링크 포함) 발송

### 시나리오 2: 과정별 멘트 업데이트
- **상황**: 새 기수 모집 시작, 과정 어필 포인트가 바뀜
- **행동**: 관리 화면에서 해당 과정의 어필 텍스트와 앞/뒤 고정 멘트 수정
- **결과**: 다음 날부터 바뀐 멘트가 자동 반영된 엑셀 출력

### 시나리오 3: 내일배움카드 미발급자 독려 문자
- **상황**: 지원자 중 내일배움카드 발급이 안 된 경우 (노션에 "아니오" 표시)
- **행동**: AI가 현실 제약(행정 절차 지연) 사유로 분석
- **결과**: 카드 발급 절차 안내에 초점을 맞춘 맞춤 독려 문자 멘트가 자동 생성

## 6. MVP 범위

### 6.1 포함 기능

| # | 기능 | 왜 MVP에 필수인가 |
|---|---|---|
| 1 | CSV 업로드 & 2일차 필터링 | 분석 대상 자동 선별 (특이사항에서 합격 안내일 추출) |
| 2 | Claude AI 이탈 사유 분석 | 서비스의 핵심 가치, 이게 없으면 기존과 동일 |
| 3 | 멘트 자동 조합 (앞 고정 + AI 본문 + 뒤 고정) | 매니저가 바로 쓸 수 있는 완성형 스크립트 |
| 4 | 엑셀 결과 파일 다운로드 | 기존 전달 방식 유지, 매니저 학습 비용 없음 |
| 5 | 과정·멘트 관리 화면 | 담당자가 직접 텍스트 수정 가능해야 운영 지속 가능 |

### 6.2 제외 범위 (추후 고도화)

- 노션 API 자동 연동: MVP 검증 후 수동 CSV의 불편함이 확인되면 추가
- 슬랙/카카오톡 자동 전송: 엑셀 전달 방식 정착 후 검토
- 전환율 추적 대시보드: 데이터 누적 후 의미 있는 수치가 생기면 추가
- 이탈 사유 분석 컬럼 고도화: 운영 데이터 보고 기준 조정
- 계정별 로그인: 사용자 2명 이상 필요해질 때 추가

## 7. 결과물 스펙

### 엑셀 출력 컬럼

| 컬럼 | 내용 | 출처 |
|---|---|---|
| 기수 | 합격 기수 | CSV |
| 이름 | 지원자 이름 | CSV |
| 연락처 | 아웃콜 번호 | CSV |
| AI/직무 이해도 | 상/중/하 | AI 분석 |
| 과정 확신도 | 상/중/하 | AI 분석 |
| 의사결정 상태 | 즉시/고민/유예 | AI 분석 |
| 현실 제약 여부 | 시간/거리/병행/행정/없음 | AI 분석 |
| 이탈 예측 사유 | 종합 한 줄 요약 | AI 분석 |
| 권장 독려 문자 멘트 | 앞 고정 + AI 본문 + 뒤 고정 + HRD 등록 링크 | AI 조합 |

### 멘트 3단 구조

```
[앞 고정 멘트]   ← 과정별로 담당자가 관리 화면에서 설정
[AI 개인화 본문] ← 이탈 사유 + 과정 어필 텍스트 기반 자동 생성
[뒤 고정 멘트]   ← 과정별로 담당자가 관리 화면에서 설정
[HRD 등록 링크] ← 고정 삽입
```

## 8. AI 분석 판단 기준

| 분석 항목 | 낮음/있음 판단 기준 |
|---|---|
| AI/직무 이해도 낮음 | 지원동기가 막연하거나 "취업이 급해서" 수준, 직무명·기술스택 언급 없음 |
| 과정 확신도 낮음 | 과정 선택 이유 설명 못하거나 다른 과정과 비교 언급 있음 |
| 현실 제약 있음 | 취업 병행·시간 제약·거리·내일배움카드 미발급 언급 있음 |
| 의사결정 유예 | 위 3가지 중 2개 이상 해당 또는 인터뷰에서 망설임 표현 있음 |

## 10. 운영 주의사항

- 노션 특이사항 컬럼에 합격 안내 날짜를 `06.15 합격 안내` 형식으로 **팀 내 표준화** 필수
- 형식이 달라지면 2일차 자동 필터가 오작동 가능 → 파싱 실패 행은 "확인 필요" 시트에 별도 출력
- 과정별 어필 텍스트는 **200자 이내** 권장 (초과 시 멘트 품질 저하 가능)

## 다음 단계 입력 요약본 (PRD Handoff Pack)

### 1. 한 줄 정의
합격 안내 후 2일차 HRD 미등록자의 이탈 사유를 Claude AI로 분석하고, 과정별 맞춤 독려 문자 멘트(HRD 등록 링크 포함)가 담긴 엑셀을 매일 오전 제공하는 내부 웹 도구

### 2. 잠긴 결정
- 타겟: 운영 담당자 1명 (추후 2~5명 확장 가능)
- 데이터 입력: 노션 CSV 수동 업로드
- AI 엔진: Claude API (claude-haiku-4-5)
- 결과물: 엑셀 파일 다운로드
- 멘트 구조: 앞 고정 + AI 개인화 본문 + 뒤 고정 (3단)
- 분석 항목 4가지: AI직무이해도 / 과정확신도 / 의사결정상태 / 현실제약
- 분석 대상 조건: 합격 안내 후 2일차 + 최종결과 = 합격안내 상태
- 접속 방식: 단일 비밀번호 (추후 계정별 로그인)

### 3. 미해결 / 추후 결정
- 서비스명 가칭 ("HRD 전환 어시스턴트") → PRD 전 확정 권장
- 이탈 사유 분석 컬럼 고도화 기준 → 운영 데이터 누적 후 결정
- 노션 API 자동 연동 → MVP 검증 후 검토

### 4. MVP 핵심 기능 목록
1. CSV 업로드 & 2일차 자동 필터링
2. Claude AI 이탈 사유 분석 (4가지 항목)
3. 멘트 자동 조합 (3단 구조)
4. 엑셀 결과 파일 다운로드
5. 과정·멘트 관리 화면 (어필 텍스트 + 앞뒤 고정 멘트)
