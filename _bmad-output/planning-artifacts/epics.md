# HRD 전환 어시스턴트 - Epic Breakdown

## Overview

This document provides the complete epic and story breakdown for HRD 전환 어시스턴트, decomposing the requirements from the PRD, FRD (UX Design), and TRD (Architecture) into implementable stories.

## Requirements Inventory

### Functional Requirements

FR1: 단일 비밀번호 인증 — 사용자가 비밀번호를 입력하면 서버가 검증하고, 성공 시 서명된 쿠키 세션을 발급한다 (8시간 유지)
FR2: 신청자 CSV 업로드 — 드래그 앤 드롭 또는 파일 선택 버튼으로 신청자 CSV를 업로드할 수 있다
FR3: 인터뷰 평가표 CSV 업로드 — 드래그 앤 드롭 또는 파일 선택 버튼으로 인터뷰 평가표 CSV를 업로드할 수 있다
FR4: CSV 필수 컬럼 즉시 검증 — 파일 업로드 직후 필수 컬럼 존재 여부를 검증하고, 누락된 컬럼명을 구체적으로 표시한다
FR5: 과정 선택 드롭다운 — DB에 등록된 과정 목록을 드롭다운으로 제공한다; 과정 없을 때 "먼저 과정을 등록해주세요" 안내와 관리 화면 링크를 표시한다
FR6: 두 CSV 이름 컬럼 기준 자동 매칭 — 신청자 CSV와 인터뷰 평가표 CSV를 이름 컬럼 기준으로 자동 매칭하고, 매칭 실패 행은 별도 표시한다
FR7: Claude AI 이탈 사유 분석 — 지원자 1명씩 순차로 4가지 항목(AI/직무이해도, 과정확신도, 의사결정상태, 현실제약 여부)을 분석한다
FR8: SSE 실시간 진행률 스트림 — 분석 중 "N명 중 M번째 분석 중" 진행 현황을 SSE로 클라이언트에 실시간 전송한다
FR9: AI 개인화 멘트 4단 조합 — 앞 고정 멘트 + AI 개인화 본문 + 뒤 고정 멘트 + HRD 등록 링크를 자동 조합한다
FR10: 엑셀 파일 다운로드 — 분석 결과를 HRD_독려문자_{과정명}_{날짜}.xlsx 형식으로 즉시 다운로드한다
FR11: 로컬스토리지 결과 저장 — 분석 결과를 브라우저 로컬스토리지에 최근 1회분만 보관한다
FR12: 재접속 시 결과 자동 복원 — 브라우저를 닫았다 재접속해도 로컬스토리지의 이전 결과를 자동으로 복원해 결과 화면을 표시한다
FR13: 로컬스토리지 덮어쓰기 및 삭제 — 새 CSV 업로드 시 자동 덮어쓰기되며, "결과 삭제" 버튼으로 수동 초기화할 수 있다
FR14: 과정별 설정 CRUD — 과정명, 앞 고정 멘트, 과정 설명(200자 이내), 뒤 고정 멘트를 저장·수정할 수 있다
FR15: 과정 추가 — 새 과정을 등록할 수 있다 (과정명 중복 시 에러 표시)
FR16: 비밀번호 5회 실패 잠금 — 5회 연속 실패 시 30초 잠금, 잠금 중 안내 메시지 표시
FR17: 분석 실패 행 처리 — API 호출 실패 시 해당 행은 "분석 실패"로 표시하고, 나머지 지원자 분석은 계속 진행한다
FR18: 고정 멘트 비어있을 때 확인 팝업 — 선택 과정의 앞/뒤 고정 멘트가 비어있을 때 "그대로 진행하시겠습니까?" 팝업을 표시한다
FR19: 같은 날 재분석 시 덮어쓰기 확인 팝업 — 로컬스토리지에 당일 결과가 있을 때 "이전 결과를 덮어씁니다" 확인 팝업을 표시한다
FR20: 분석 취소 — 분석 중 취소 링크를 클릭하면 "취소하면 처음부터 다시 해야 합니다" 확인 팝업 후 CSV 업로드 화면으로 이동한다

### NonFunctional Requirements

NFR1: 분석 완료 시간 — 지원자 5명 기준 30~60초 이내 분석 완료
NFR2: API 키 보안 — Claude API 키는 서버 환경변수(.env)에만 저장; 프론트엔드 코드에 절대 노출 금지
NFR3: 개인정보 비저장 — 업로드된 CSV(이름·연락처 등)는 분석 완료 후 서버 메모리에서 즉시 삭제; DB에 개인정보 저장 금지
NFR4: API 비용 — Claude API(Haiku) 월 비용 200원 미만 (하루 5명 기준)
NFR5: 분석 성공률 — 95% 이상 (분석 실패 행 / 전체 행으로 매일 측정)
NFR6: 세션 보안 — 세션은 itsdangerous 서명된 쿠키로 구현; 8시간 유지
NFR7: 로그 관리 — RotatingFileHandler 10MB 상한, 백업 3개 보관
NFR8: 단순 실행 — 더블클릭 배치 파일(.bat Windows)로 서버 시작 가능; 비개발자 담당자 친화

### Additional Requirements

- 모듈러 모놀로식 구조: auth 모듈(비밀번호 인증·세션), analysis 모듈(CSV·Claude·엑셀), course 모듈(과정 설정 CRUD) — 3개 모듈, 서버 1개
- 프론트엔드: React + Vite + TypeScript SPA (Next.js 아님); shadcn/ui + Tailwind CSS
- 폼 검증: React Hook Form + Zod
- 백엔드: Python 3.11+ + FastAPI
- CSV 파싱: pandas; 엑셀 생성: openpyxl (실패 행 빨간색 서식 포함)
- AI 호출: anthropic 공식 Python SDK
- SSE 구현: FastAPI async generator 패턴 필수 (동기 함수 사용 시 스트림 차단됨)
- DB: SQLite + SQLAlchemy ORM + Alembic 마이그레이션 (COURSES 테이블 1개)
- 서빙: 프론트엔드 빌드 파일을 FastAPI가 정적 파일로 직접 서빙 (서버 프로세스 1개)
- 환경변수: CLAUDE_API_KEY, APP_PASSWORD, SESSION_SECRET_KEY를 .env에 저장; .gitignore에 .env 포함, .env.example 커밋
- 실행 스크립트: start.bat (개발), deploy.bat (빌드+재시작) 제공
- 가상환경 활성화: 배치 파일에 venv 활성화 코드 포함 필수 (TRD 리스크 MEDIUM-1 대응)
- SQLite DB 경로: DB_PATH를 .env에서 관리, 기본값 ~/hrd-data/hrd.db

### UX Design Requirements

UX-DR1: S-001 비밀번호 입력 화면 구현 — 로고 + 마스킹 비밀번호 필드 + 확인 버튼; 5가지 상태 (Empty/Loading/Error/Success/Disabled) 전체 구현

UX-DR3: S-003 분석 진행 중 화면 구현 — 로딩 스피너 + "N명 중 M번째 분석 중" 텍스트 + 예상 완료 시간 + 분석 취소 링크(하단 소형); 5가지 상태 전체 구현

## Epic 1: 프로젝트 기반 & 접속 인증

담당자가 비밀번호를 입력해 도구에 처음 접속할 수 있다. React+Vite+TypeScript 프론트엔드, Python+FastAPI 백엔드, SQLite DB, .env 환경변수 구성, 더블클릭 .bat 실행 스크립트가 모두 작동하는 상태가 된다.

### Story 1.1: 프로젝트 초기 설정

As a 개발자,
I want React+Vite+TypeScript 프론트엔드와 FastAPI+SQLite 백엔드가 하나의 서버에서 실행되는 기반 구조를,
So that 이후 모든 기능을 이 구조 위에 구현할 수 있다.

**Acceptance Criteria:**

**Given** start.bat을 더블클릭하면
**When** 가상환경이 활성화되고 빌드된 프론트엔드를 FastAPI가 정적 파일로 서빙하면
**Then** 브라우저에서 http://localhost:8000에 접속 시 React 앱이 로드된다
**And** deploy.bat 실행 시 npm run build 후 서버가 재시작된다

**Given** .env.example 파일이 저장소에 포함되어 있으면
**When** 개발자가 .env를 새로 만들어야 할 때
**Then** .env.example의 모든 키(APP_PASSWORD, SESSION_SECRET_KEY, CLAUDE_API_KEY, DB_PATH)를 참조해 생성할 수 있다
**And** .env는 .gitignore에 포함되어 커밋되지 않는다

**Given** 앱이 시작될 때 필수 환경변수가 없으면
**When** FastAPI 서버가 부팅되면
**Then** 명확한 에러 메시지와 함께 서버가 종료된다

**Given** Alembic 마이그레이션을 실행하면
