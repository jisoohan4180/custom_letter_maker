# Tutorial Manager 서비스 정의서

> 버전: v2.0 MVP (1단계 4주 범위 기준)
> 작성일: 2026-05-21
> 대상: 비개발자 1인 · Claude Code 바이브코딩 · 4주 출시

---

## 0. Executive Summary

**한 줄 정의**: 교육 매니저가 훈련생 명단을 한 번 업로드하면 개인 코드가 자동 발급되고, 훈련생은 코드 입력만으로 픽셀 게임 스타일 튜토리얼 화면에 진입해 미션을 제출하며, 매니저는 한 화면에서 전체 진행도를 색상으로 확인할 수 있는 내부 운영 자동화 도구

**1단계 MVP 범위 (4주)**:
① 명단 업로드 → 개인 코드 자동 발급
② 코드 입력 → 픽셀 미션 화면 → 이미지 포함 미션 제출
③ 매니저 진행도 대시보드 (🔴🟡🟢 색상 + 행 상세 펼침)

**필요 자원**: 비개발자 1명 / 4주 / 외부 비용 $0

**기대 효과**:
- 기수당 코드 생성 수작업: 30명 × 5분 = 150분 → 0분
- 진행도 확인: 30개 페이지 개별 클릭 → 대시보드 1화면

---

## 1. 타깃 고객

### 1.1 1순위 — 매니저
- 플레이데이터 아카데미 내부 매니저 1~3명
- 담당 기수 2개, 기수당 최대 30명 (오프라인)
- 현재 행동: 노션 페이지 수동 복사 → 링크 개별 전달 → 진행도 하나씩 확인

### 1.2 2순위 — 훈련생
- 개강 1주차 훈련생 (코딩 비전공자 포함)
- 새 경험: 공용 QR 스캔 또는 링크 접속 → 코드 입력 → 픽셀 게임 스타일 튜토리얼 진입

---

## 2. 핵심 문제

매 기수마다 동일한 수작업 3종이 반복된다.

1. **생성**: 훈련생별 노션 페이지를 1개씩 만들어 링크 전달 (30명 × 5분 = 150분)
2. **수정**: 오탈자 발생 시 30개 페이지 개별 수정
3. **모니터링**: 진행도 확인을 위해 페이지를 1개씩 클릭

---

## 3. 1단계 MVP 기능 (4주 범위)

| # | 기능 | 포함 이유 |
|---|---|---|
| 1 | 훈련생 명단 CSV 업로드 → 개인 코드 자동 발급 | 식별 기반. 없으면 나머지 불가 |
| 2 | 코드 입력 → 개인 튜토리얼 홈 진입 (공용 QR 또는 링크) | 훈련생 접근 방식 |
| 3 | 픽셀 게임 스타일 미션 화면 + 텍스트/이미지 제출 | 핵심 훈련생 경험 |
| 4 | 위험군 색상 대시보드 (🔴🟡🟢) + 행 상세 펼침 | 이탈자 조기 식별 |
| 5 | 코드 비활성화 (철회자 처리) | 대시보드 노이즈 방지 |

### 미션 콘텐츠 제작 방식
Notion에 있는 기존 튜토리얼 내용을 **참고해** 서비스 내 DB에 미션 콘텐츠를 별도 작성한다. Notion API 직접 연동은 하지 않는다. 매니저는 관리자 화면 탭1에서 미션 제목·본문·이미지 필요 여부를 직접 입력한다.

---

## 4. 단계별 로드맵

### 1단계 (지금 만드는 것 · 4주)
코드 발급 → 미션 화면 → 이미지 제출 → 진행도 대시보드

### 2단계 (첫 기수 운영 후)
- Discord DM 자동 발송 + 발송 성공/실패 목록 + 개별 재발송
- CSV 재업로드 중복 처리
- 이미지 자동 삭제 (Vercel Cron)

### 3단계 (데이터 쌓인 후)
- 기수 기록 탭 (종료 기수 아카이브)
- 예측 적중률
- 관리자 로그인 고도화 (Supabase Auth)

---

## 5. 제외 결정 (1단계에서 안 함)

| 기능 | 제외 이유 |
|---|---|
| Discord DM 자동 발송 | Bot 설정·서버 권한·Intent 활성화 — 비개발자 4주 범위 초과 |
| 외부 자동화 툴 (Make.com 등) | 의존성 증가, 이미 이전 버전에서 제거 확정 |
| Notion API 연동 | 인증·캐싱·API 제한 복잡성, 1단계 불필요 |
| 이미지 자동 삭제 | 운영 직전 Cron 설정이므로 2단계 |
| 기수 기록·예측 적중률 | 첫 기수 데이터 없는 상태에서 불필요 |

## 수정 이유 요약

**줄인 것**: Discord DM 자동 발송(서버 권한·Bot Intent 설정이 비개발자에게 난이도 높음), 기수 기록 탭, 예측 적중률 자동 계산, Notion API 연동

**뒤로 미룬 이유**: Discord 연동은 Bot 생성·서버 초대·Intent 활성화가 모두 필요해 4주 비개발자 범위를 초과한다. 기수 기록·예측 적중률은 첫 기수 데이터가 쌓여야 의미 있다.

**1단계에서 먼저 만들 것**: "명단 업로드 → 코드 발급"과 "코드 입력 → 픽셀 미션 화면 → 제출 → 대시보드"를 순서대로 완성하면 핵심 가치(수작업 제거 + 이탈자 조기 식별)를 즉시 검증할 수 있다.
