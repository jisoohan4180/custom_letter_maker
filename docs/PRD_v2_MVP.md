## 1. 한 문단 요약

Tutorial Manager는 플레이데이터 아카데미 매니저가 매 기수마다 반복하던 노션 수작업 3종을 단일 웹앱으로 대체한다. 1단계에서는 세 가지를 완성한다. 첫째, 매니저가 CSV를 업로드하면 훈련생별 개인 코드가 자동 발급된다. 둘째, 훈련생은 공용 링크(또는 QR)에서 코드를 입력해 픽셀 게임 스타일의 개인 튜토리얼 화면으로 진입하고 미션을 텍스트 또는 이미지로 제출한다. 셋째, 매니저는 대시보드 한 화면에서 30명의 진행도를 색상(🔴🟡🟢)으로 확인하고 행을 펼쳐 미션별 제출 현황을 본다.

Discord DM 자동 발송은 2단계에서 추가한다. 1단계에서 코드 전달은 매니저가 목록을 복사해 수동으로 발송한다. 개발자 1명(비개발자·Claude Code) 4주, 외부 비용 $0.

## 2. 왜 만드는가

### 2.1 현재 어떻게 일하는가
기수 시작마다 매니저는 노션 페이지를 30개 수동 복사 → 링크를 하나씩 전달 → 진행도를 30개 페이지 클릭으로 확인한다.

### 2.2 무엇이 문제인가
반복성. 기수가 바뀌어도 작업이 달라지지 않는다. 오탈자 수정 시 30개 페이지를 모두 열어야 하고, 이탈 위험자를 찾으려면 하나씩 클릭해야 한다.

### 2.3 어떻게 바꾸는가
CSV 업로드 한 번 → 코드 자동 발급 → 매니저가 목록에서 코드를 복사해 전달 → 훈련생 코드 입력 → 픽셀 미션 화면 → 제출 → 대시보드에서 3초 만에 위험군 식별.

## 4. 사용자 시나리오

### 시나리오 1: 기수 시작 — 코드 발급 후 수동 전달
매니저가 훈련생 명단 CSV(이름, 전화 뒷 4자리)를 관리자 화면에 업로드한다. 시스템이 즉시 코드(예: 박민준6789)를 생성하고 중복을 검사한다. 매니저는 생성된 코드 목록을 복사해 Discord에서 수동으로 전달한다. (Discord 자동 발송은 2단계)

### 시나리오 2: 훈련생 접속 — 코드 입력 후 미션 수행
박민준은 교실 QR 또는 링크에서 코드를 입력한다. 픽셀 게임 스타일의 개인 튜토리얼 홈 화면이 열리고 미션 목록이 퀘스트처럼 나타난다. 미션을 클릭하면 상세 모달이 열리고 텍스트 입력 또는 이미지 업로드로 제출한다. 완료 시 "CLEAR!" 연출 후 카드에 ✅가 붙는다.

### 시나리오 3: 진행도 확인 — 색상으로 위험군 즉시 식별
기수 D+3일, 매니저가 대시보드를 연다. 완료 미션이 2개 이하인 훈련생이 🔴로 표시된다. 행을 클릭하면 미션별 제출 현황이 인라인으로 펼쳐진다. 매니저는 해당 훈련생에게 선제 연락한다.

## 5. 무엇을 만드는가

### 5.1 화면 구조 (1단계 4개 화면)

```
/enter (S-001 코드 입력)
  └─ 유효 코드 → /t/:code (S-002 튜토리얼 홈)
       └─ 미션 클릭 → 모달 S-003 미션 상세
                        └─ 완료 → 닫힘, 카드 ✅

/admin/cohorts/:id (S-004 기수 상세)
  ├─ 탭1: 세팅 (명단 업로드, 코드 목록, 비활성화)
  └─ 탭2: 대시보드 (색상, 행 펼침)
```

> 기수 목록 화면(/admin/cohorts)은 1단계에서 단순 목록으로만 구현한다. 종료 기수 이탈 수 입력·적중률 계산은 3단계.

### 5.2 핵심 기능

**코드 발급 (매니저)**
CSV 업로드 → 이름+뒷4자리 코드 자동 생성 → 중복 자동 감지 → 목록 표시 + 복사 버튼. Discord 자동 발송 없이 수동 전달.

**미션 콘텐츠 (매니저가 직접 입력)**
Notion 원본을 참고해 DB에 미션 콘텐츠를 직접 작성한다. 탭1 미션 편집 에디터에서 제목·본문·이미지 필요 여부를 입력 후 저장. 저장 즉시 훈련생 화면에 반영. Notion API 미사용.

**미션 제출 (훈련생)**
텍스트 답변 또는 인증샷(최대 10MB) 업로드 → Supabase Storage 저장 → 완료 처리. 완료된 미션은 재제출(교체) 가능.

**대시보드 (매니저)**
기수 전체 진행 현황. D+3일 이후 완료 수 기준 색상 자동 표시. 행 클릭 시 미션별 완료 여부 인라인 펼침. 코드 비활성화 버튼 포함.

### 5.3 콘텐츠 관리 방식 확정
- 1단계: DB 저장 + 관리자 에디터(실시간 Markdown 렌더링)
- Notion API 연동: 없음
- git push 배포 방식: 없음 (DB 저장으로 대체)

### 5.4 데이터 모델 (개요)

| 테이블 | 핵심 필드 |
|---|---|
| cohorts | id, name, starts_at, expires_at |
| trainees | id, cohort_id, access_code, is_active, created_at |
| missions | id, cohort_id, order_index, title, content, requires_image |
| submissions | id, trainee_id, mission_id, content, image_url, submitted_at |

| 주차 | 작업 |

## 수정 이유 요약

**줄인 것**: Discord DM 자동 발송, 기수 기록 탭, 예측 적중률, Notion API, git push 콘텐츠 배포 방식

**뒤로 미룬 이유**:
- Discord 연동은 Bot 생성·서버 초대·Intent 활성화 3단계가 모두 필요해 비개발자 4주 범위를 초과한다.
- 기수 기록·적중률은 첫 기수 데이터가 없으면 사용할 수 없다.
- Notion API는 인증·속도 제한·캐싱 설계가 필요해 1단계 복잡도를 크게 높인다.

**1단계에서 먼저 만들 것**: 명단 업로드 → 코드 발급 → 코드 입력 → 픽셀 미션 화면 → 이미지 제출 → 대시보드 색상 확인. 이 흐름이 완성되면 핵심 가치(수작업 제거 + 이탈자 식별)가 즉시 검증된다.
