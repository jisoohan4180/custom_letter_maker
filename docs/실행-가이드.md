# HRD 전환 어시스턴트 — 실행 가이드

## A. 회사 PC에서 쓰기 (설치 없음, 비개발자용) ⭐

Python·Node 설치가 전혀 필요 없습니다. **빌드된 폴더만 복사**하면 됩니다.

### 1) 배포본 만들기 (개발 PC에서 1번만)
```powershell
./scripts/build-exe.ps1
```
→ `release\HRD-Assistant\` 폴더가 생깁니다. (수 분 소요, 용량 수십~백 MB)

### 2) 회사 PC로 옮기기
- `release\HRD-Assistant\` **폴더를 통째로** USB나 공유드라이브로 회사 PC에 복사.

### 3) 회사 PC에서 설정 (1번만)
- 폴더 안의 `.env.example` 을 복사해 이름을 `.env` 로 바꾸고, 메모장으로 열어 값 채우기:
  ```
  APP_PASSWORD=접속비밀번호
  SESSION_SECRET_KEY=아무_긴_랜덤문자열
  CLAUDE_API_KEY=sk-ant-...   (console.anthropic.com 발급)
  DB_PATH=./hrd.db
  ```

### 4) 실행
- `HRD-Assistant.exe` **더블클릭** → 검은 창이 뜨고 잠시 후 **브라우저가 자동으로 열립니다**.
- 안 열리면 브라우저에서 직접 `http://localhost:8000` 입력.
- 같은 사무실 다른 직원은 `http://<이 PC의 IP>:8000` 으로 접속 가능.
- **종료**: 검은 창을 닫으면 됩니다.

> ⚠️ 회사 보안정책이 서명 안 된 .exe 실행을 막으면 IT팀에 실행 허용을 요청하세요.
> ⚠️ AI 분석은 인터넷 + 유효한 `CLAUDE_API_KEY` 가 있어야 동작합니다.
> ℹ️ 등록한 과정·멘트는 그 PC의 `hrd.db` 에 저장됩니다 (PC마다 따로).

## B. 개발 PC에서 소스로 실행 (개발자용)

```powershell
# 최초 1회: venv + 패키지 + 프론트 빌드는 start.bat 이 자동 처리
./start.bat
```
또는 직접:
```powershell
./venv/Scripts/python.exe launcher.py   # 마이그레이션 + 브라우저 + 서버
```
→ `http://localhost:8000`

프론트 코드를 바꿨으면 `./deploy.bat` (프론트 재빌드 후 실행) 또는 `npm run build` 후 실행.

## C. 개인정보 보호 (참고)

- 지원자 **이름·연락처는 외부 AI(Claude)로 전송되지 않습니다** (가명처리). 분석에는 인터뷰 평가 내용만 사용하고, 엑셀에는 실명이 그대로 들어갑니다.
- 개인정보(CSV)는 분석 후 서버 메모리에서 삭제되며 DB에 저장되지 않습니다.
- 이 도구는 **사내 PC 실행**을 전제로 설계되었습니다. 외부 클라우드 배포는 개인정보 국외 이전 등 별도 검토가 필요합니다.
