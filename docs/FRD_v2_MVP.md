## 1. 한 문단 요약

1단계 FRD는 4개 화면을 다룬다. 훈련생 화면 3개(코드 입력·튜토리얼 홈·미션 상세 모달)와 매니저 화면 1개(기수 상세 2탭)다. 미션 콘텐츠는 DB에 저장하며 탭1 실시간 에디터로 편집한다. Discord 자동 발송·기수 목록 이탈 수 입력·예측 적중률은 이 문서에 포함하지 않는다.

---

## 2. 화면 지도

```
/enter (S-001 코드 입력)
  └─ 유효 코드 → /t/:code (S-002 튜토리얼 홈)
       └─ 미션 카드 클릭 → 모달 (S-003 미션 상세)
                            └─ CLEAR! 2초 → 닫힘

/admin (매니저 비밀번호 진입)
  └─ /admin/cohorts (단순 목록)
       └─ /admin/cohorts/:id (S-004 기수 상세)
            ├─ 탭1: 세팅 + 미션 편집
            └─ 탭2: 대시보드
```

## 4. 화면별 상세 명세

### 4.1 S-001 코드 입력

```
[ AI 캠프 튜토리얼 ]

오프라인 자료에서 받은 코드를 입력하세요

[ 박민준6789           ]
[ 입장하기              ]

(에러) 코드를 다시 확인해주세요
(에러) 튜토리얼이 종료된 코드입니다
(에러) 사용할 수 없는 코드입니다
```

| 사용자 행동 | 시스템 반응 |
|---|---|
| 유효 코드 입력 | 픽셀 "입장!" 연출 → S-002 이동 |
| 없는 코드 | "코드를 다시 확인해주세요" |
| 만료 코드 (14일 경과) | "튜토리얼이 종료된 코드입니다" |
| 비활성 코드 | "사용할 수 없는 코드입니다" |
| 기존 세션 보유 | 코드 입력 없이 S-002 자동 이동 |

**설계 결정**: 기수 시작일 기준 14일 후 코드 자동 만료. 만료와 비활성화는 에러 메시지로 구분한다.

### 4.3 S-003 미션 상세 모달

모달은 **보기 모드**와 **편집 모드** 두 상태를 가진다. 미완료 미션은 처음부터 편집 모드로 열린다. 이미 제출한 미션은 보기 모드로 열리고, 수정하기 버튼을 누르면 편집 모드로 전환된다.

**보기 모드 (제출 완료 후 재진입)**

```
어두운 배경 ──────────────────────────
│  [ LEVEL UP! ]                 ✕  │
│                                    │
│  미션 안내 본문 (DB content 렌더링)  │
│  ─────────────────────────────     │
│  (📷 인증샷 미션인 경우)
|  [ 🖼️ ] [ 🖼️ ] [ 🖼️ ]  ← 썸네일   
|  ───────────────────────────── 
|  제출한 텍스트 내용이 여기 표시됩니다│
│     
│                                    │
│                     [ ✏️ 수정하기 ]|
──────────────────────────────────────
```

**편집 모드 (미완료 첫 진입 또는 수정하기 클릭 후)**

```
어두운 배경 ──────────────────────────
│  [ LEVEL UP! ]                 ✕  │
│                                    │
│  미션 안내 본문 (DB content 렌더링)  │
│  ─────────────────────────────     │
│  [ 텍스트 입력창                   ] │
│  [ (기존 내용 채워진 상태)         ] │
│                                    │
│  (📷 인증샷 미션인 경우)             │
│  [ 🖼️ ✕ ] [ 🖼️ ✕ ] [ + 추가 ]     │
│  최대 5장 · 장당 10MB 이하          │
│                                    │
│  [ 저장 ]   [ 취소 ]               │
│                                    │
│  (저장 후 첫 제출) ★ CLEAR! ★ → 2초 │
──────────────────────────────────────
```

| 사용자 행동 | 시스템 반응 |
|---|---|
| 미완료 미션 클릭 | 편집 모드로 바로 오픈 |
| 완료 미션 클릭 | 보기 모드로 오픈 |
| 수정하기 클릭 | 보기 모드 → 편집 모드 전환 (기존 내용 채워진 상태) |
| 취소 클릭 | 편집 모드 → 보기 모드 복귀 (변경 내용 버림) |
| 저장 (첫 제출) | DB 저장 → CLEAR! 2초 → 모달 닫힘 → 카드 ✅ |
| 저장 (재제출) | DB 업데이트 → 보기 모드 전환 ("수정 완료" 표시) |
| 이미지 추가 (+ 버튼) | 파일 선택 → 썸네일 즉시 미리보기 |
| 이미지 개별 삭제 (✕) | 해당 썸네일 제거 (저장 전까지 Storage 삭제 안 함) |
| 5장 초과 추가 시도 | "+ 추가" 버튼 비활성화, "최대 5장" 안내 |
| 10MB 초과 파일 | "10MB 이하 파일만 업로드할 수 있습니다" |
| ESC / 배경 / ✕ | 모달 닫힘 (편집 중이면 "저장하지 않고 닫을까요?" 확인) |
| URL 직접 접속 | S-002 로드 후 해당 모달 자동 오픈 |

**설계 결정**:
- `requires_image`가 false이면 이미지 업로드 영역 자체가 표시되지 않는다.
- 이미지는 저장 버튼을 눌러야 Storage에 업로드된다. 취소 시 미업로드 상태로 버려진다.
- 기존 이미지 ✕ 삭제는 저장 시 일괄 처리한다 (Storage 파일 삭제 + DB 레코드 삭제).
- 텍스트가 없는 미션(requires_image only)은 텍스트 입력창을 표시하지 않는다.

### 4.4 S-004 기수 상세

#### 탭1 — 세팅 + 미션 편집

**A. 명단 관리 영역**

```
AI캠프 3기   2026.05.19 시작 ~ 2026.06.02 만료

[ 명단 업로드 (CSV) ]

이름  [            ]  뒷4자리  [      ]  [ 개별 추가 ]

이름      코드              활성
박민준    박민준6789        ON ●
김철수    김철수1234        ON ●
이영희    이영희5678        OFF ○

[ 코드 전체 복사 ]
```

| 사용자 행동 | 시스템 반응 |
|---|---|
| CSV 업로드 | 파싱 → 코드 생성 → 목록 표시 |
| 중복 코드 발생 | 해당 행 붉은 강조 + "수동 조정 필요" 메시지 |
| 개별 추가 | 중복 검사 → 목록 추가 |
| 활성화 토글 | is_active 즉시 반영 |
| 코드 전체 복사 | 클립보드에 "이름: 코드" 형식 목록 복사 |

**설계 결정**: Discord 자동 발송 버튼은 이 화면에 없다. 1단계에서 코드 전달은 "전체 복사" 버튼으로 수동 처리한다.

---

**B. 미션 편집 영역**

```
──────────────────────────────
[ 미션 편집 ]

MISSION 01
제목  [ 환경 설정하기                    ]
내용  (실시간 Markdown 에디터 — 입력 즉시 렌더링)
📷 인증샷 필요  [ OFF ○ ]
[ 저장 ]

MISSION 02
...

[ + 미션 추가 ]
```

| 사용자 행동 | 시스템 반응 |
|---|---|
| 미션 내용 입력 | 실시간 Markdown 렌더링 (우측 프리뷰) |
| 인증샷 토글 | requires_image 즉시 반영 |
| 저장 클릭 | DB 저장 → 훈련생 화면 즉시 반영 |
| 미션 추가 | 빈 미션 행 추가 |
| 미션 순서 변경 | 드래그 앤 드롭 (order_index 업데이트) |

**설계 결정**: Notion 원본을 참고해 매니저가 이 에디터에 직접 입력한다. Notion API 연동 없음. 썸네일 이미지는 1단계에서 선택 사항(없으면 기본 픽셀 이미지 표시).

#### 탭2 — 대시보드

```
이름      진행도             위험도   페이지
박민준    3/7  ■■■□□□□     🟡       🔗
김철수    1/7  ■□□□□□□     🔴       🔗
이영희    7/7  ■■■■■■■     🟢       🔗

(행 클릭 시 인라인 펼침)
└ MISSION 01  ✅ 2026.05.20
  MISSION 02  ✅ 2026.05.21
  MISSION 03  □  —
  ...
```

**위험도 색상 기준**

| 색상 | 조건 | 기준 |
|---|---|---|
| 색상 없음 | D+3일 이전 | 기수 시작일 기준 |
| 🔴 위험 | D+3일 이후, 완료 0~2개 | — |
| 🟡 진행 중 | D+3일 이후, 완료 3~5개 | — |
| 🟢 완료 | D+3일 이후, 완료 6~7개 | — |

> D+3 기준은 기수별 시작일(cohorts.starts_at)이 아닌 훈련생 개인 합류일(trainees.created_at) 기준이다.

| 사용자 행동 | 시스템 반응 |
|---|---|
| 행 클릭 | 미션별 완료 여부 + 완료 일시 인라인 펼침 |
| 🔗 클릭 | /t/:code 새 탭으로 열기 |
| 코드 비활성화 | 해당 행에서 토글 → is_active OFF |

## 수정 이유 요약

**줄인 것**: Discord DM 발송 버튼, 발송 성공/실패 목록, 기수 목록의 종료 기수 이탈 수 입력, 예측 적중률 계산, 기수 기록 탭

**충돌 해소**:
- FRD v1.4 S-004에 "예측 적중률 자동 계산" 로직이 잔존해 있었으나 서비스 정의서·PRD에서 삭제 확정된 항목이므로 제거
- PRD/TRD는 Markdown 파일 기반, FRD v1.4는 DB+관리자 에디터 방식으로 서로 달랐으나 DB+관리자 에디터 방식으로 통일

**뒤로 미룬 이유**: Discord 자동 발송은 Bot 생성·서버 초대·Intent 권한 설정이 연쇄 의존성을 가진다. 비개발자가 4주 안에 핵심 흐름을 완성하려면 이 의존성을 1단계에서 제거해야 한다. 코드 전체 복사 버튼으로 수동 전달이 가능하므로 운영상 큰 지장이 없다.

**1단계에서 먼저 만들 것**: S-001(코드 입력) → S-002(픽셀 홈) → S-003(미션 제출) → S-004 탭1(명단 업로드+미션 편집) → S-004 탭2(대시보드). 이 흐름이 완성되면 1단계 MVP 완성이다.
